原型链与继承机制详解

🎯 本文目标：深入理解 JavaScript 原型链的工作原理

什么是原型链？

在 JavaScript 中，每个对象都有一个内部属性 [[Prototype]]，它指向该对象的原型对象。

const obj = {}
console.log(Object.getPrototypeOf(obj)) // Object.prototype

当我们试图访问一个对象的属性时，如果对象本身没有这个属性，JavaScript 引擎会去它的原型对象中查找。

原型链的尽头

所有普通对象的原型链最终都会指向 Object.prototype，而 Object.prototype 的原型是 null。

实现继承的几种方式

  • 原型链继承：直接修改原型指向
  • 构造函数继承：使用 call 或 apply
  • 组合继承：结合前两者
  • ES6 Class：语法糖，本质仍是原型链

在现代开发中，我们通常推荐使用 ES6 的 class 关键字来处理面向对象编程，因为它更加直观且易于维护。
