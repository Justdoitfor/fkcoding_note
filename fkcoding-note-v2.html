原型链与继承机制详解
📅 2026-04-23 ✍️ fkcoding ⏱ 约 6 分钟阅读 📝 1,247 字 ✓ 已发布
🎯

本文目标：深入理解 JavaScript 原型链的工作原理，掌握原型式继承、构造函数继承和 ES6 Class 继承的实现差异与最佳实践。

什么是原型链？

在 JavaScript 中，每个对象都有一个内部属性 [[Prototype]]，它指向该对象的原型对象。当我们访问一个属性时，JavaScript 引擎会沿着原型链向上查找，直到找到该属性或到达链的顶端（null）。

const animal = {
  type: 'Animal',
  describe() {
    return `I am a ${this.type}`;
  }
};

const dog = Object.create(animal);
dog.type = 'Dog';

console.log(dog.describe()); // "I am a Dog"
console.log(dog.__proto__ === animal); // true
💡

现代 JavaScript 推荐使用 Object.getPrototypeOf(obj) 代替 __proto__，后者是非标准的遗留属性。

构造函数与 prototype

每个函数都有一个 prototype 属性，它是一个对象，包含所有该构造函数实例共享的方法。

new 操作符原理

当使用 new 调用构造函数时，JavaScript 引擎执行以下步骤：

  • 创建一个新的空对象
  • 将新对象的 [[Prototype]] 设置为构造函数的 prototype
  • 以新对象为 this 执行构造函数体
  • 如果构造函数返回对象则使用该对象，否则返回新对象

ES6 Class 继承

特性构造函数方式ES6 Class
语法函数式声明式，更清晰
提升函数提升不提升（TDZ）
严格模式可选强制启用
静态方法手动挂载static 关键字
私有字段约定 _ 前缀#field 真私有
⚠️

Class 是语法糖，底层依然基于原型链。理解原型链机制是掌握 Class 行为的基础。

最佳实践建议

在现代 JavaScript 开发中，优先使用 ES6 Class 语法，它更具可读性并支持现代 IDE 的静态分析。避免直接操作 __proto__，改用 Object.create() 或 Object.setPrototypeOf()。
